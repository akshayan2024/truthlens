04 — Getting the Transcript

Three approaches failed.
One worked.

✕ window.ytInitialPlayerResponse

Isolated world

Content scripts can't read page JS variables. The data is right there — completely unreachable.

✕ InnerTube API

CORS + session tokens

YouTube's internal API blocks extension contexts and requires constantly-changing session tokens.

✕ DOM scraping

Panel must be open

The transcript panel only renders if the user opens it manually. DOM structure breaks with YouTube A/B tests.

✓ Supadata API

Server-side transcript fetch

Third-party API fetches YouTube transcripts server-side, bypassing CORS entirely. Returns timestamped segments — perfect for timestamp linking in the sidebar.

06 — The Agentic Loop

The AI drives every search

01
Pick a claim
AI reads the transcript, identifies the next checkable factual claim
02
Craft a search query
AI writes its own specific query — not us hardcoding one. Gets 3 Tavily results tagged [CREDIBLE] or [UNVERIFIED].
03
Search again if needed
If results are poor or off-topic, AI searches with a different query — up to 25 iterations total
04
Categorize from evidence
Only now does the AI assign a verdict — strictly from search results, never training knowledge
// Provider-agnostic agent loop
while (i < MAX_ITERS) {

  const res = await callAI(messages, tools);

  // No tool calls → agent done
  if (!res.tool_calls) break;

  // Batch searches in parallel
  const results = await Promise.all(
    res.tool_calls.map(tc =>
      searchTavily(tc.query)
    )
  );
  messages.push(results);
}

return parseJSON(finalMessage);

09 — Key Challenges

The hard parts

Chrome MV3 Isolated World

Content scripts can't read page JS

The transcript lives in window.ytInitialPlayerResponse — completely inaccessible from a content script. Required a server-side transcript API.

YouTube SPA Navigation

Page doesn't reload between videos

YouTube is a single-page app. Used yt-navigate-finish events + a 2s heartbeat to re-inject the button whenever YouTube removes it.

Training Data Contamination

AI confidently wrong from memory

Even with Tavily results, the AI would default to training knowledge. Required explicit prompt rules + hard code enforcement post-response.

Sidebar Timing Race

FACT_CHECK_START lost on cold load

Sidebar iframe wasn't ready when content.js sent the first message. Fixed with a SIDEBAR_READY handshake and 1.5s fallback timeout.

10 — Key Learnings

What I learned

01
Agentic beats rigid pipelines
Letting the AI choose its own search queries produces dramatically better results than hardcoded queries. The model knows what it needs to know.
02
Source provenance matters more than source count
One result from reuters.com is worth more than 10 from unknown blogs. Tagging sources as credible vs unverified explicitly changes verdict quality.
03
Chrome extensions are deceptively complex
Isolated worlds, service workers, CSP headers, SPA navigation, iframe sandboxing, MV3 restrictions — each is simple, but they interact in surprising ways.
04
Design language communicates credibility
A fact-checker that looks like a chatbot feels less trustworthy than one that looks like a reference tool. Design signals intent before users read a word.
